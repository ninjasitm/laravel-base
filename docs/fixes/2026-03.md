## [2026-03-28] - PHP 8.4 Deprecation Cleanup

### Fixed

- **Issue**: First-party PHP files emitted PHP 8.4 deprecations during linting, and a small set of legacy syntax patterns were incompatible with current PHP.
  - **Root cause**: Multiple methods used typed parameters with `= null` but without explicit nullable types. One method declared an optional parameter before a required one. One listener still used reversed `implode()` argument order, and one helper used an unparenthesized nested ternary.
  - **Solution**: Replaced implicit nullable signatures with explicit nullable types, made `getRRule()` accept an empty request data array by default to preserve call order, corrected `implode()` ordering, parenthesized the nested ternary, and added PHPUnit regression tests for both lint-time compatibility and message preparation.
  - **Files modified**: `Api/src/Http/Controllers/Auth/*SocialAuthController.php`, `Api/stubs/tests/TestCase.stub.php`, `Content/src/Configuration/ManagesApiOptions.php`, `Content/src/Listeners/BaseNotifyOfActivity.php`, `Content/src/Notifications/*.php`, `Content/src/Repositories/FileRepository.php`, `Content/src/Traits/*.php`, `Helpers/src/VisibilityHelper.php`, `Testing/src/TestCase.php`, `tests/Unit/BaseNotifyOfActivityTest.php`, `tests/Unit/PhpCompatibilityTest.php`
  - **Status**: ✅ Fixed

### Impact

- 🔧 **Technical**: Repository source and stubs lint cleanly under PHP 8.4 without deprecation or parse errors.
- 🎯 **User Experience**: Test runs and package maintenance are no longer obscured by PHP compatibility noise.
- 🚨 **Breaking Changes**: None.

## [2026-03-29] - PR Follow-Up Runtime Regression Fixes

### Fixed

- **Issue**: Follow-up PR review surfaced runtime bugs in automation broadcasts and relation syncing that were not visible in static diagnostics.
  - **Root cause**: Both automation event classes built private channels with `reduce()` while mutating an outer array by value, `NewReaction` never populated its recipient IDs, `SyncsRelations` used an impossible empty-collection guard plus an invalid `syncWithoutDetatching()->sync(...)` chain, and `SyncsNotificationPreferences` inverted its user/team model checks.
  - **Solution**: Rebuilt broadcast channel lists with collection `map()->all()`, derived reaction recipients from the reactable owner and commenters, fixed the empty-input guard and `syncWithoutDetaching($data)` call, corrected notification preference owner assignment, and added focused regression tests for each path.
  - **Files modified**: `Content/src/Events/{NewComment,NewReaction}.php`, `Content/src/Traits/{SyncsRelations,SyncsNotificationPreferences}.php`, `tests/Unit/{AutomationEventBroadcastTest,SyncsRelationsTest,SyncsNotificationPreferencesTest}.php`

### Impact

## [2026-03-29] - Tag Script Release Notes

### Fixed

- **Issue**: The release tag script used a generic annotation message instead of the latest master commit message, so annotated tags did not carry forward the actual release summary.
  - **Root cause**: `bump-git-tag.sh` hard-coded the tag annotation text and created a lightweight tag before attempting to create an annotated one.
  - **Solution**: Changed the script to read the latest `master` commit message once, reuse it for version bump markers and annotated tag release notes, allow optional `--message` and `--message-file` overrides for custom release notes, fall back to the master commit message when no override content is provided, target the `master` commit hash explicitly, and create a single annotated tag.
  - **Files modified**: `bump-git-tag.sh`, `tests/Unit/BumpGitTagScriptTest.php`
  - **Status**: ✅ Fixed

### Impact

- 🔧 **Technical**: Release tags now inherit the current master commit message as their annotation body and no longer attempt a conflicting lightweight-plus-annotated tag sequence.
- 🎯 **User Experience**: Git tag release notes now reflect the actual merge or release commit summary automatically, while still allowing manual overrides through `--message` or `--message-file` when needed.

## Patterns & Learnings

### Common Issues Discovered

- PHP 8.4 deprecates implicitly nullable typed parameters; use explicit `?Type` or `Type|null` in first-party code and stubs.
- Lint-based regression coverage catches compile-time deprecations that normal runtime tests may never execute.

### Best Practices Established

- Include stubs and test helpers in PHP-version compatibility sweeps, not just runtime source files.
- Keep a dedicated compatibility test in the suite when supporting newer PHP versions.
- Use `autoload-dev` stubs for optional integrations when the package intentionally avoids hard runtime dependencies.
- Add behavior regressions for review-found runtime issues even when diagnostics are already clean.
- Tag automation should create one annotated tag and derive release notes from the commit being published unless an explicit `--message` or `--message-file` override is passed.
